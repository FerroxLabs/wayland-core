01

Onboarding · connect a provider — net-new surface

There's no setup wizard today — this is redesign scope, grounded on real primitives: the Anthropic OAuth device flow (OAuthManager — real), per-provider API-key auth, init_config(), and the --doctor Ollama probe. Key-prefix detection is a small new nicety; the engine does not validate keys against /v1/models and has no folder-trust concept, so both were cut. The Orbit mark is Wayland's brand glyph; orange-on-near-black is the Hearth Palette.

02

Workspace · idle — new TUI shell

The steady state. The status bar shows model (claude-sonnet-4-6 — the real id), cwd, branch, context meter, and the session mode. The mode badge is one of the engine's three real SessionModes: Default / Auto-edit / Yolo — cycled by ⇧Tab via the SetMode command. Flat color, single accent, no shadows or gradients (brand §07) — exactly what a terminal renders.

03

Agent · live — grounded

Tool names are verbatim from wcore-tools (Read, Grep, Edit). The diff-as-approval is the redesign's core move — the host renders the diff from the Edit tool's old_string/new_string args (the engine's Edit returns only "replaced N occurrences"). The 3-option approval maps 1:1 to real protocol commands: ToolApprove{Once} / ToolApprove{Always} (category edit) / ToolDeny{reason}. The compat diff is real engine code.

04

Sub-agents · live monitor — grounded on the Spawn tool

A live monitor of an in-flight Spawn call. The main agent spawned 4 parallel sub-agents (max 5) — each a named agent from .wayland/agents/*.yaml — with a real live feed (ChannelSink relay), turn and token counts. They run independently with no shared state — Spawn forbids coordination, so there are no progress bars, no dependencies, no "blocked" state. Results return to the main agent when all settle. A persistent orchestrator (lifecycle, worktree merge) would need wcore-swarm wired into the CLI — a roadmap item.

05

Command Palette — new dispatcher

Today's REPL has only /quit and /exit — the palette is net-new. Every command shown is grounded on a real capability: /rewind → RollbackTool file snapshots (files only — conversation-history rewind doesn't exist), /resume → SessionManager, /repomap → wcore-repomap, /replay → wcore-replay. ~14 commands map to existing flags; user-invocable skills (wcore-skills) extend the set.

06

Plan Review · plan mode — real engine feature

Plan mode is real: the agent calls EnterPlanMode and is restricted to read-only tools until it presents the change set — "writes nothing until you approve" is genuine engine behavior. Entered by asking Wayland to plan (a /plan command would be net-new sugar) — not Shift+Tab, which cycles approval modes. Approve exits plan mode and runs; the "$EDITOR" edit option was cut (no such hook).

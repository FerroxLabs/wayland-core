# Wayland Core Frontier Candidate v2

## What This Is

Wayland Core is a Rust 2021 multi-provider agent engine with a standalone CLI/TUI and a versioned host protocol consumed by Wayland Desktop. This milestone finishes the existing Frontier program from F20 onward, then extends it with operator-complete continuous agency, gateway, ecosystem, migration, multimodal, and release-proof contracts.

## Core Value

Deliver a best-in-class cross-platform agent that stays simple in ordinary use, bounded and auditable under managed policy, crash-complete, transactionally delegated, operator-complete, and honestly proven through the packaged product.

## Business Context

- **Customer**: Individuals and teams that need a capable daily agent with optional enterprise-grade control and evidence.
- **Revenue model**: Wayland product value and differentiation; this Core milestone does not define pricing.
- **Success metric**: One exact candidate satisfies every active requirement with zero unresolved findings, required native-platform evidence, packaged standalone/host journeys, and evidence-backed peer positioning.
- **Strategy notes**: The winning position is Hermes-grade persistent agency and OpenClaw-grade product completeness on Wayland's stronger authority, recovery, transaction, and evidence spine.

- **Proof routing**: Authoritative Linux Cargo proof runs on `hetzner-dsm:/root/wayland`; native Windows and macOS evidence is required at declared gates.
- **Security**: Managed authority cannot widen; dangerous local execution is explicit, bounded, visible, and policy-controlled.
- **Compatibility**: Preserve CLI/TUI/host-protocol behavior or version migrations explicitly; Desktop must never consume an uncommitted dirty Core checkout.

- **Findings**: BLOCKER, CRITICAL, HIGH, MEDIUM, and LOW findings are repaired or rejected with evidence before acceptance; zero unresolved findings remain.

Phase 20 does not claim cryptographic resistance to a same-UID actor that can rewrite Git history, scripts, and Git-private state; that requires an external signer or isolated control plane and is outside this pure-GSD execution milestone.
